dardcor は Windows で直接実行できますが、より快適に使うには [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) の利用をおすすめします。WSL は dardcor の機能とスムーズに連携する Linux 環境を提供します。

:::tip[WSL を使う理由]
WSL を使うと、ファイルシステム性能、ターミナルサポート、dardcor が依存する開発ツールとの互換性が向上します。
:::

---

## セットアップ

<Steps>

1. **WSL をインストールする**

   まだの場合は、Microsoft 公式ガイドを使って [WSL をインストール](https://learn.microsoft.com/en-us/windows/wsl/install) します。

2. **WSL で dardcor をインストールする**

   WSL の準備ができたら WSL のターミナルを開き、[インストール方法](/docs/) のいずれかで dardcor をインストールします。

   ```bash
   curl -fsSL https://dardcor-code.vercel.app//install | bash
   ```

3. **WSL から dardcor を使う**

   プロジェクトディレクトリに移動し（Windows ファイルは `/mnt/c/` や `/mnt/d/` などからアクセス）、dardcor を実行します。

   ```bash
   cd /mnt/c/Users/YourName/project
   dardcor
   ```

</Steps>

---

## デスクトップアプリ + WSL サーバー

dardcor デスクトップアプリを使いつつ、サーバーは WSL で動かしたい場合は次の手順です。

1. **WSL でサーバーを起動する**

   外部接続を許可するため、`--hostname 0.0.0.0` を付けて起動します。

   ```bash
   dardcor serve --hostname 0.0.0.0 --port 4096
   ```

2. **デスクトップアプリを接続する**

   `http://localhost:4096` に接続します。

:::note
環境によって `localhost` が使えない場合は、WSL 側で `hostname -I` を実行して IP アドレスを確認し、`http://<wsl-ip>:4096` に接続してください。
:::
:::caution
`--hostname 0.0.0.0` を使う場合は、`dardcor_SERVER_PASSWORD` を設定してサーバーを保護してください。

```bash
dardcor_SERVER_PASSWORD=your-password dardcor serve --hostname 0.0.0.0
```

:::

## Web クライアント + WSL

Windows で Web 利用を快適にするには:

1. **PowerShell ではなく WSL ターミナルで `dardcor web` を実行する**

   ```bash
   dardcor web --hostname 0.0.0.0
   ```

2. **Windows のブラウザからアクセスする**

   `http://localhost:<port>` にアクセスします（URL は dardcor が表示します）。

WSL から `dardcor web` を実行すると、適切なファイルシステムアクセスとターミナル統合を維持したまま、Windows ブラウザから利用できます。

---

## Windows ファイルへのアクセス

WSL からは `/mnt/` ディレクトリ経由で Windows ファイルにアクセスできます。

- `C:` drive → `/mnt/c/`
- `D:` drive → `/mnt/d/`
- そのほかのドライブも同様です

例:

```bash
cd /mnt/c/Users/YourName/Documents/project
dardcor
```

:::tip
よりスムーズに使うには、リポジトリを WSL のファイルシステム（例: `~/code/`）にクローンまたはコピーして、そこで dardcor を実行することをおすすめします。
:::

---

## ヒント

- Windows ドライブ上のプロジェクトでも、dardcor は WSL で実行するとファイルアクセスがスムーズです
- dardcor と一緒に VS Code の [WSL 拡張機能](https://code.visualstudio.com/docs/remote/wsl) を使うと統合的な開発フローを構築できます
- dardcor の設定とセッションは WSL 環境内の `~/.local/share/dardcor/` に保存されます
